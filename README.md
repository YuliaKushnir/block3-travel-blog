## Welcome to block3-travel-blog

При створенні проекту використано предметну область з блоків 1-2: "Блог подорожей", яка містить дві сутності: Користувач та Допис. 

За основу взято наданий шаблон, в якому створено окрему гілку "blog-implementation". Код додано з використанням технологій, які вже використовувались в шаблоні та з якими ознайомились під час лекцій. 

Для використання без бекенду використовуються мок-дані, збережені до redux-store.

Оскільки концепція застосунку передбачає, що переглядати список дописів можуть всі користувачі (сторінка default), а створювати, видаляти, редагувати допис будь-якого користувача (сторінка secret) може лише адмін, для доступу до повного функціоналу необхідно авторизуватись, використовуючи:
- логін: admin
- пароль: admin


### До сторінок default і secret додано основний функціонал згідно завдання: 

#### Роут /default
- перегляд списку дописів (з декількома основними полями) на сторінці з пагінацією;
- фільтрування списку за категоріями, країнами, рейтингом (фільтри не скидаються);
- видалення допису (кнопка з'являється при наведенні курсора на картку, при успіху/помилці виникає відповідне сповіщення);
- при натисненні на картку - перехід на сторінку з детальною інформацією;
- в зручному місці кнопка для переходу на сторінку для створення нового допису;

#### Роут /secret/:id
- перегляд детальну інформацію дописів;
- кнопка "Назад" перенаправляє на "Список дописів" (фiльтр та пагiнацiя залишаються такими ж)
- кнопка-іконка "Олівець" відкриває форму для редагування деяких полів.

#### Роут /secret/new
- створення нового допису (валідація полів, отримання сповіщення про успіх або помилку через глобальний Redux‑notification);

#### Роут /secret/:id
- редагування існуюго допису (валідація полів, отримання сповіщення про успіх або помилку);

#### Інтерфейс користувача 
простий, інтуїтивно зрозумілий

#### Інтернаціоналізацію застосовано до обох сторінок

## Встановлення залежностей

```sh
npm install
```

## Запуск у режимі розробки

```sh
npm start
```

## Збірка для продакшн

```sh
npm run build
```

## Зібрати Docker-image

```sh
docker build -t tb-frontend .
```

## Запуск контейнера в мережі shared-net

```sh
docker run -d --name frontend -p 3050:80 --network shared-net tb-frontend

```





## Використані технології
- **React** (hooks, компоненти).
- **Redux** (стан постів, користувачів, повідомлень).
- **react‑jss** (стилізація через тему).
- **Material‑UI (MUI)** (компоненти: Dialog, TextField, Autocomplete).
- **JavaScript (ES6+)**.
